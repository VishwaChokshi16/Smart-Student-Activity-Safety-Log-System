# Smart Student Activity & Safety Log Automation System

## 📌 Project Overview

The **Smart Student Activity & Safety Log Automation System** is a UNIX-based automation project developed using **Bash shell scripting**. The purpose of this system is to automatically monitor and record student activities in a centralized log system.

In educational environments, manually tracking student activities can be inefficient and prone to errors. This project provides a **simple and automated solution** that logs important actions such as student login events, activity tracking, and safety alerts.

The system uses **UNIX shell scripts, log files, and automation techniques** to ensure that student activities are properly recorded and monitored.

---

## 🎯 Objectives

* Automate student activity monitoring.
* Maintain structured log files for student activities.
* Track student login information.
* Generate safety alerts for unusual activities.
* Reduce manual effort in maintaining activity records.

## 📂 Project Structure

```
Smart_Student_Monitor
│
├── student_login.sh        # Records student login activities
├── activity_logger.sh      # Logs student activities
├── safety_monitor.sh       # Detects unusual activities and generates alerts
├── view_logs.sh            # Displays stored activity logs
│
├── logs
│   └── activity.log        # Stores all student activity records

└── README.md               # Project documentation
```

## ▶️ How to Run the Project

### Step 1: Open Terminal

Navigate to the project directory.

```
cd Smart_Student_Monitor
```

### Step 2: Run the Scripts

Run each script using bash.

```
bash student_login.sh
```

```
bash activity_logger.sh
```

```
bash safety_monitor.sh
```

```
bash view_logs.sh
```

---

## 📝 Example Workflow

1. A student logs into the system.
2. The **student_login.sh** script records the login activity.
3. The **activity_logger.sh** script stores activity information in a log file.
4. The **safety_monitor.sh** script checks for unusual or suspicious activities.
5. All activities are stored in **activity.log** for future reference.
